1 · The bug in your own code

This is your memstore.go, as of last night:

func (s *MemStore) Get(ctx context.Context, code string) (Link, error) {
    return s.links[code], nil
}

Ask it for a code that doesn't exist and it returns Link{} — empty Code, empty URL — and says nil: "everything went fine." In a real shortener that's a redirect to an empty URL, and nobody finds out until a user does. The map read gave you the zero value (warm-up question 3), and the function turned a missing link into a fake success.

The fix has two halves, and they're today's whole lesson: detect the miss (the comma-ok read), and report it as a value the caller can check (a sentinel error).

2 · error is just an interface

Go has no exceptions. error is an ordinary interface from Day 1's world:

type error interface {
    Error() string
}

An error is a value you return, pass, compare, and store — the same as an int or a Link (Rob Pike, "Errors are values"). You make one with errors.New or fmt.Errorf, and the calling convention is the pair return you've been writing since yesterday: (Link, error), where exactly one of the two is meaningful.

Coming from languages with exceptions this feels manual. The payoff: the error path is visible in the code — every if err != nil is a place a reviewer can see a decision being made, instead of an invisible jump to a handler three files away.

3 · Sentinel errors and errors.Is

A sentinel error is a package-level error declared once, so callers can recognize it:

// in package link
var ErrNotFound = errors.New("link: not found")

You already wrote one! But you named it ErrorNotFound — the convention is a terse Err prefix: ErrNotFound. Interviewers notice this stuff; part of today's practice is the rename.

Callers check for it with errors.Is — not ==:

if errors.Is(err, link.ErrNotFound) {
    // show a friendly 404, don't page anyone at 3am
}

Why not ==? Because of what section 4 does to errors on their way up the call stack.

4 · Wrapping: add context on the way up

A bare "link: not found" arriving at the top of a server tells you nothing about which request, which code, which layer. So each layer wraps what it got with fmt.Errorf and the %w verb:

l, err := store.Get(ctx, code)
if err != nil {
    return Link{}, fmt.Errorf("resolve %q: %w", code, err)
}

Now the message reads resolve "missing": link: not found — an error chain — and errors.Is still finds ErrNotFound through the wrapping, which plain == never would (Go blog: Working with Errors in Go 1.13). That's the whole division of labor:

LayerJob
where the error is bornreturn a checkable value (ErrNotFound)
every layer it passes throughadd context with %w, return — don't log-and-continue
ONE layer at the topdecide: 404? retry? log? crash? — handle once

One more tool for your interview belt: when callers need data out of an error (not just identity), you define a custom error type and dig it out with errors.As(err, &target) — same chain-walking, but by type. Read it today, use it on Day 5 when Postgres errors carry codes.

Step 3 — a wrapping layer. In main.go, above main, add the resolver (this is given — read the %w line until it makes sense):

// resolve looks up a code and wraps any failure with context about THIS call.
// The %w verb keeps the original error inside the new one, findable by errors.Is.
func resolve(ctx context.Context, store link.Store, code string) (link.Link, error) {
    l, err := store.Get(ctx, code)
    if err != nil {
        return link.Link{}, fmt.Errorf("resolve %q: %w", code, err)
    }
    return l, nil
}

Step 4 — handle once, at the top. Replace the Get/print part of main (keep the Save) with a loop that hits both the happy and the missing path:

for _, code := range []string{"go", "missing"} {
    l, err := resolve(ctx, store, code)
    switch {
    case errors.Is(err, link.ErrNotFound): // matches even through the %w wrap
        fmt.Printf("no link for %q — try another code\n", code)
    case err != nil:
        fmt.Fprintln(os.Stderr, "unexpected:", err)
        os.Exit(1)
    default:
        fmt.Println(l.Code, "→", l.URL)
    }
}

Add "errors" and "os" to main.go's imports (gopls/your editor likely does it for you).

func TestLoggingStoreDelegates(t *testing.T) {

        t.Fatalf("save was not delegated to the inner store: %+v", fake.saved)
