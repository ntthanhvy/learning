Data Wrangling · Reference

Glossary

Every jargon term used in the lessons, in one place. Tiếng Việt column is dev-style — how Vietnamese devs actually say it, not textbook translation.

TermTiếng Việt (dev-style)In software
data wranglingxử lý / làm sạch dữ liệu thô (hay giữ nguyên "wrangle")Cleaning, restructuring, and enriching raw data into a usable shape — the hands-on craft data jobs test in interviews.
ETLETL (giữ nguyên) — biến đổi xong mới nạp vào khoExtract–Transform–Load: pull data from sources, clean/reshape it in a processing step (often Python/pandas), then load the finished result into the destination.
ELTELT (giữ nguyên) — nạp thô vào warehouse trước, biến đổi sauExtract–Load–Transform: raw data lands in the warehouse first and is transformed there, usually with SQL/dbt. The modern default for analytics pipelines.
DataFrameDataFrame (giữ nguyên) — bảng 2 chiều, giống table SQLpandas' central object: a table of named, typed columns with labeled rows. Your SQL table instincts apply to it almost line for line.
SeriesSeries (giữ nguyên) — một cột của DataFrameOne column of a DataFrame: a 1-dimensional labeled array where every value shares one dtype. df["amount"] returns one.
dtypedtype (giữ nguyên) — kiểu dữ liệu của CẢ một cộtThe data type of an entire column (int64, float64, object/string, datetime64…). Per column, not per cell — like a typed column in Postgres.
NumPyNumPy (giữ nguyên) — thư viện mảng nền tảng, pandas chạy trên nóThe foundational library for fast typed arrays in Python. Each DataFrame column is a NumPy array — that's where pandas' speed comes from.
vectorizedvectorized (giữ nguyên) — thao tác cả cột một lượt, không loopApplying an operation to a whole column/array in one call, executed in compiled C, instead of looping over elements in the Python interpreter. The core pandas methodology.
boolean maskmask (hay nói tắt) — mảng True/False để lọc hàngA True/False Series aligned to the rows; indexing a DataFrame with it keeps only the True rows. pandas' WHERE clause: df[df["amount"] > 100].
NaNgiữ nguyên "NaN" — giá trị thiếu, so sánh == luôn ra Falsepandas'/NumPy's marker for missing floating-point data. Detected with isna() — never with ==, since NaN != NaN by IEEE float rules.
NaTgiữ nguyên "NaT" — NaN dành riêng cho cột datetimepandas' missing-value marker specifically for datetime columns — the datetime64 equivalent of NaN.
Indexgiữ nguyên "Index" — nhãn hàng, tách biệt với các cột dữ liệuThe labels attached to a DataFrame's rows, separate from the data columns. .loc looks up by this label; .iloc looks up by raw position regardless of it.
RangeIndexgiữ nguyên "RangeIndex" — index mặc định 0,1,2,3..., không mang ý nghĩa nghiệp vụThe default pandas Index after loading data: 0, 1, 2, 3… — just positions, carrying no business meaning until you set_index a real key.
coerce (errors="coerce")giữ nguyên "coerce" — ép kiểu, lỗi thì thành NaN chứ không crashA conversion mode (pd.to_numeric/pd.to_datetime) that turns unparseable values into NaN/NaT instead of raising — the pandas equivalent of SQL's TRY_CAST.
dropnagiữ nguyên "dropna" — bỏ hàng thiếu dữ liệuRemoves rows (or columns) containing missing values; subset=[...] limits the check to specific columns. SQL cousin: WHERE col IS NOT NULL.
fillnagiữ nguyên "fillna" — điền giá trị thay cho thiếuReplaces missing values with a given value (or a computed one, like a median). SQL cousin: COALESCE(col, value).
mediangiữ nguyên "median" — giá trị giữa khi sắp xếp, ít bị outlier kéo lệchThe middle value of a sorted column — half the values lie above it, half below. A single extreme outlier barely moves it, unlike the mean, which is why it's a safer default fill value.
flag columnflag column — cột đánh dấu, thường là booleanAn extra boolean column (e.g. amount_was_missing) recording a fact about a row — here, whether a value was originally missing before it got filled, so that information isn't silently lost.
groupby / split-apply-combinegiữ nguyên 'split-apply-combine' — chia nhóm, tính từng nhóm, rồi gộp kết quả lạiThe three-step model behind every groupby: SPLIT rows into groups by a key, APPLY a function to each group independently, COMBINE the per-group results back into one Series/DataFrame. groupby() alone only does the split — nothing computes until an aggregation is chained on.
named aggregationgiữ nguyên 'named aggregation' — đặt tên cột kết quả trực tiếp trong .agg()The .agg(new_col=("source_col", "func")), or .agg(new_col="func") on an already-selected column, form — names each output column directly instead of pandas' uglier default auto-generated names from a plain dict.
merge / joingiữ nguyên 'merge' — hàm gộp 2 DataFrame theo cột khóa, tương đương JOIN trong SQLpandas' function for combining two DataFrames row-wise on a shared key column — the direct equivalent of a SQL JOIN. pd.merge(left, right, on="key", how="inner"/"left"/"right"/"outer") controls both the match column and which unmatched rows survive.
indicator / _mergegiữ nguyên 'indicator' / '_merge' — cột đánh dấu hàng khớp ở bên nàoindicator=True on a merge adds a _merge column valued "both", "left_only", or "right_only" per row — the direct way to find rows present on only one side of a join.
suffixes (_x / _y)giữ nguyên 'suffixes' — hậu tố tự thêm khi 2 bảng trùng tên cộtWhen both merged DataFrames share a non-key column name, pandas silently renames both with _x (left) and _y (right) instead of erroring — pass suffixes=(...) to name them clearly yourself.
long formatgiữ nguyên 'long format' — mỗi hàng 1 quan sát, mỗi cột 1 biếnA table shape where each row is one observation and each column is one variable — the natural output shape of SQL SELECT/GROUP BY and every DataFrame used through Lesson 5.
wide formatgiữ nguyên 'wide' — dàn 1 giá trị thành nhiều cột riêngThe same data laid out with a distinct value spread across separate columns instead of separate rows — a spreadsheet pivot table or monthly report's usual shape.
pivot_tablegiữ nguyên 'pivot_table' — hàm dàn bảng long thành wideA pandas DataFrame method reshaping long data into wide: one row per index value, one column per distinct value in columns, cells filled from values via aggfunc.
meltgiữ nguyên 'melt' — hàm dàn bảng wide về lại longA pandas DataFrame method reshaping wide data back into long: id_vars stay as columns, every other column becomes a (var_name, value_name) pair of rows.
window functionwindow function (giữ nguyên) — tính theo nhóm nhưng KHÔNG gộp hàng, khác GROUP BYA SQL feature computing a value across a set of related rows (a PARTITION) without collapsing them into one row per group, unlike GROUP BY. rank() and cumsum() chained after groupby() are pandas' equivalent.
rank()giữ nguyên 'rank()' — đánh số thứ hạng, giống RANK()/ROW_NUMBER() OVER trong SQLA pandas Series method assigning each value its rank within an ordering — the direct analog of SQL's RANK()/ROW_NUMBER() OVER (...) window functions, computed per group when chained after groupby().
rank tie-breaking (method=)giữ nguyên 'method=' — cách xử lý khi 2 giá trị bằng nhaurank()'s method= argument decides what happens when two values tie: "first" breaks ties by row order (like ROW_NUMBER()), "min"/"dense" give tied rows the same rank (like RANK()/DENSE_RANK()).
top N per groupgiữ nguyên 'top N per group' — lấy k dòng đứng đầu mỗi nhómA common interview pattern: within each group, keep only the highest (or lowest) k rows by some ordering — solved in pandas via rank() <= k or sort_values().groupby().head(k), and in SQL via ROW_NUMBER() OVER (PARTITION BY ...) filtered to <= k.
cumsum()giữ nguyên 'cumsum()' — tổng dồn, tương đương SUM() OVER (...)A running/cumulative sum: each row's output is the sum of itself and every prior row (in the current order) — the pandas analog of SQL's SUM() OVER (PARTITION BY ... ORDER BY ...) window frame. Trusts row order only — no ORDER BY argument of its own.
method chainingmethod chaining (giữ nguyên) — nối liên tiếp các method, không đặt tên biến trung gianWriting a sequence of DataFrame method calls back-to-back, each one operating on the result of the last, instead of assigning each intermediate step to its own variable name.
lambdalambda (giữ nguyên) — hàm không tên, viết gọn trong một dòngPython's syntax for a small, unnamed, inline function — lambda d: d + 1 is a function taking d and returning d + 1. Used inside assign() to name the in-progress DataFrame at that exact point in the chain.
.pipe().pipe() (giữ nguyên) — nhét một hàm tùy ý vào giữa chainA DataFrame method that calls a given function with the DataFrame as its argument and returns that function's result, letting a custom multi-line transformation stay inside a chain instead of breaking it into a separate reassigned step.
