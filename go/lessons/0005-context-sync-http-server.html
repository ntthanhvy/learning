1 · context: one cancellation signal, threaded everywhere

Every Store method has taken a ctx context.Context since Day 1 — today it stops being decoration. context.Context is the same idea as yesterday's time.After in checkAll's select, generalized: instead of one hardcoded timer, any caller — an HTTP middleware, a test, another goroutine — can set a deadline and every function downstream that checks ctx.Done() obeys it, without knowing who set it or why.

ctx, cancel := context.WithTimeout(parent, 2*time.Second)
defer cancel()   // ALWAYS defer cancel — releases the timer even if you return early

select {
case r := <-resultCh:
    return r, nil
case <-ctx.Done():
    return zero, ctx.Err()   // context.DeadlineExceeded or context.Canceled
}

That select is exactly yesterday's shape — ctx.Done() is just a channel, same as time.After's. The difference is composability: an HTTP server can set one timeout in a middleware, and every store call, every query, every goroutine three layers down shares it automatically, because the same ctx value gets passed down. That's why the rule is "ctx first parameter, always" — it's not style, it's how cancellation actually propagates.

2 · Goroutine leaks — the bug context exists to prevent

Warm-up question 2 named it without a term: a goroutine leak is a goroutine still blocked on a channel operation that will never happen. Yesterday's orphaned workers weren't leaked — they finished on their own and their send just found nobody listening (a `select` with only one receive attempt, not blocked forever). The real leak pattern is subtler: a goroutine send with no timeout and no cancellation path at all. The fix is always the same: give every blocking operation a way out via ctx.Done() in its own select. Rule of thumb for interviews: "never start a goroutine without knowing how it stops."

One stdlib-adjacent tool worth knowing by name (not required in linkshort, which stays stdlib-only): golang.org/x/sync/errgroup bundles "launch N goroutines, cancel all of them the moment one fails, wait for the rest" into a few lines — the production version of yesterday's WaitGroup fan-out when any worker's error should stop the others.

3 · ServeMux, 1.22-style

Go 1.22 added method- and wildcard-matching to the standard router — no framework needed:

mux := http.NewServeMux()
mux.HandleFunc("POST /links", srv.handleCreate)
mux.HandleFunc("GET /{code}", srv.handleRedirect)
mux.HandleFunc("GET /{code}/stats", srv.handleStats)

{code} is a wildcard segment; the handler reads it with r.PathValue("code"). Method prefixes ("GET ", "POST ") mean the mux itself rejects the wrong verb with 405 — no manual if r.Method != ... anywhere (Routing Enhancements for Go 1.22).

4 · Middleware is still just a decorator

Nothing new in shape here — a func(http.Handler) http.Handler wraps a handler exactly the way LoggingStore wrapped a Store on Day 1: one job, delegate the rest.

func loggingMiddleware(log *slog.Logger, next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        next.ServeHTTP(w, r)                 // delegate first (or after — logging needs "after")
        log.Info("request", "method", r.Method, "path", r.URL.Path, "took", time.Since(start))
    })
}

Today's second middleware — the timeout — is today's TODO (section 5 practice): it does the same wrapping, but calls context.WithTimeout and swaps the request's context before delegating.

5 · Graceful shutdown

Killing a server mid-request drops whatever it was doing. The idiomatic shape: catch the OS signal, stop accepting new connections, let in-flight ones finish, then exit.

ctx, stop := signal.NotifyContext(context.Background(), syscall.SIGINT, syscall.SIGTERM)
defer stop()

go func() { httpServer.ListenAndServe() }()   // non-blocking: main must reach the line below

<-ctx.Done()                                    // blocks here until Ctrl-C / SIGTERM
httpServer.Shutdown(shutdownCtx)                // stop taking new conns, drain in-flight, then return

signal.NotifyContext turns an OS signal into the exact same ctx.Done() shape you've used all lesson — graceful shutdown is just one more cancellation signal, same tool, different source (Go blog: Context).

Step 2 — server.go (given, except one TODO). handleCreate and resolve reuse Day 2's wrapping pattern; hits is a plain map guarded by a mutex — you fill in the guard:

package main

import (
    "crypto/rand"
    "encoding/base64"
    "encoding/json"
    "errors"
    "fmt"
    "net/http"
    "sync"
    "time"

    "linkshort/link"
)

type server struct {
    store link.Store
    mu    sync.Mutex
    hits  map[string]int
}

func newServer(store link.Store) *server {
    return &server{store: store, hits: make(map[string]int)}
}

func newCode() (string, error) {
    buf := make([]byte, 4)
    if _, err := rand.Read(buf); err != nil {
        return "", err
    }
    return base64.RawURLEncoding.EncodeToString(buf), nil
}

type createRequest struct {
    URL string `json:"url"`
}

type createResponse struct {
    Code string `json:"code"`
    URL  string `json:"url"`
}

// handleCreate: POST /links {"url": "..."} -> {"code": "...", "url": "..."}
func (s *server) handleCreate(w http.ResponseWriter, r *http.Request) {
    var req createRequest
    if err := json.NewDecoder(r.Body).Decode(&req); err != nil || req.URL == "" {
        http.Error(w, `body must be {"url": "..."}`, http.StatusBadRequest)
        return
    }
    code, err := newCode()
    if err != nil {
        http.Error(w, "internal error", http.StatusInternalServerError)
        return
    }
    l := link.Link{Code: code, URL: req.URL, CreatedAt: time.Now()}
    if err := s.store.Save(r.Context(), l); err != nil {
        http.Error(w, "internal error", http.StatusInternalServerError)
        return
    }
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(http.StatusCreated)
    json.NewEncoder(w).Encode(createResponse{Code: l.Code, URL: l.URL})
}

// resolve wraps the store lookup with request-scoped context — Day 2's
// pattern, now carrying whatever deadline the timeout middleware set.
func (s *server) resolve(r *http.Request, code string) (link.Link, error) {
    l, err := s.store.Get(r.Context(), code)
    if err != nil {
        return link.Link{}, fmt.Errorf("resolve %q: %w", code, err)
    }
    return l, nil
}

// handleRedirect: GET /{code} -> 302 to the target URL, counts the hit.
func (s *server) handleRedirect(w http.ResponseWriter, r *http.Request) {
    code := r.PathValue("code")
    l, err := s.resolve(r, code)
    switch {
    case errors.Is(err, link.ErrNotFound):
        http.NotFound(w, r)
        return
    case err != nil:
        http.Error(w, "internal error", http.StatusInternalServerError)
        return
    }

    // TODO: s.hits[code]++ is a data race under concurrent redirects —
    // guard it: s.mu.Lock(); s.hits[code]++; s.mu.Unlock()

    http.Redirect(w, r, l.URL, http.StatusFound)
}

type statsResponse struct {
    Code string `json:"code"`
    URL  string `json:"url"`
    Hits int    `json:"hits"`
}

// handleStats: GET /{code}/stats -> {"code", "url", "hits"} (given).
func (s *server) handleStats(w http.ResponseWriter, r *http.Request) {
    code := r.PathValue("code")
    l, err := s.resolve(r, code)
    switch {
    case errors.Is(err, link.ErrNotFound):
        http.NotFound(w, r)
        return
    case err != nil:
        http.Error(w, "internal error", http.StatusInternalServerError)
        return
    }
    s.mu.Lock()
    hits := s.hits[code]
    s.mu.Unlock()
    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(statsResponse{Code: l.Code, URL: l.URL, Hits: hits})
}

Step 3 — main.go (replaces the CLI demo). Wiring is given; the shutdown block is your TODO.

package main

import (
    "context"
    "errors"
    "log/slog"
    "net/http"
    "os/signal"
    "syscall"
    "time"

    "linkshort/link"
)

func main() {
    store := link.NewMemStore()
    srv := newServer(store)

    mux := http.NewServeMux()
    mux.HandleFunc("POST /links", srv.handleCreate)
    mux.HandleFunc("GET /{code}", srv.handleRedirect)
    mux.HandleFunc("GET /{code}/stats", srv.handleStats)
