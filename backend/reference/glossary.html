Backend Foundations · Reference

Glossary

Every jargon term used in the lessons, in one place. Tiếng Việt column is dev-style — how Vietnamese devs actually say it, not textbook translation.

TermTiếng Việt (dev-style)In software
serverserver (giữ nguyên)A long-running program that listens on a port and answers requests. Not the physical machine — the process.
portport / cổng (thường giữ nguyên)A number (like 8080 or 5432) that lets one machine run many listening programs; the address within a machine.
endpointendpoint (giữ nguyên)One callable URL + method that an API exposes, e.g. GET /api/orders/{id}.
route / routingroute (giữ nguyên); routing = định tuyếnThe rule mapping a method + path pattern to the function that should handle it; routing is the act of matching.
handlerhandler (giữ nguyên) — hàm xử lý requestThe function that receives one request and produces one response. The backend equivalent of an event handler.
middlewaremiddleware (giữ nguyên) — lớp chạy trước/sau handlerCode wrapped around handlers to do cross-cutting work — logging, auth checks, timeouts — so every handler doesn't repeat it.
business logiclogic nghiệp vụ / business logicThe rules of your product ("an order over $100 ships free"), kept separate from HTTP parsing and database code.
statelessstateless (giữ nguyên) — không giữ trạng thái giữa các requestThe server keeps nothing in memory between requests; each request starts from zero. State that must survive goes to the database.
persistencepersist / lưu xuống DBStoring data so it survives after the process stops — usually meaning "write it to the database".
connection poolconnection pool (giữ nguyên) — pool kết nối DBA kept-warm set of database connections that handlers borrow and return, because opening a fresh connection per request is too slow.
serializationserialize (giữ nguyên) — chuyển object thành JSON/bytesTurning an in-memory value into bytes to send over the wire (e.g. struct → JSON); deserialization is the reverse.
schemaschema (giữ nguyên) — cấu trúc bảng của DBThe set of tables, columns, and rules defining what data can exist — the database's shape, shared by every feature.
entityentity (giữ nguyên) — thực thể, có id riêngA thing in the product with its own identity and lifecycle — it gets an id, usually its own table, and other things refer to it.
primary keyprimary key / khóa chính (hay nói tắt PK)The column that uniquely identifies one row — usually an id — which other tables point at.
foreign keyforeign key / khóa ngoại (hay nói tắt FK)A column holding another table's primary key, expressing "this row belongs to that row".
join tablejoin table / bảng trung gianA table whose rows each connect two other tables' rows — how a many-to-many relationship is stored (e.g. order_items between orders and products).
normalizationnormalization / chuẩn hóa dữ liệuStructuring tables so each fact is stored exactly once and everything else references it by key. Instinct version: one fact, one place.
denormalizationdenormalization (giữ nguyên) — cố ý lưu trùng dữ liệuDeliberately storing a fact in more than one place — a trade-off you must justify (e.g. snapshotting a price onto an order line).
API contractAPI contract (giữ nguyên) — hợp đồng giữa backend và clientThe agreed shape of an API — URLs, verbs, bodies, error format. The promise frontends build against; breaking it breaks them.
resourceresource (giữ nguyên) — "danh từ" mà API exposeA thing clients address by URL — usually an entity or collection (/orders, /orders/42). Verbs act on it; you don't make /getX endpoints.
idempotentidempotent (giữ nguyên) — gọi N lần như gọi 1 lầnSame end state whether performed once or many times, so blind retries are safe. GET/PUT/DELETE promise it; POST does not.
status codestatus code (giữ nguyên) — chữ số đầu nói ai có lỗiThe response's three-digit result. 2xx worked; 4xx the client's request was wrong (fixable by caller); 5xx the server broke (alarm).
error contracterror contract — MỘT format lỗi thống nhấtThe single JSON error shape every failing response uses, with a stable machine-readable code clients branch on — the API's sentinel errors.
authenticationauthentication (giữ nguyên) — xác minh "bạn là ai"Proving who is making a request — distinct from authorization, which decides what they're allowed to do once known.
cookiecookie (giữ nguyên)A small piece of data the browser stores and automatically re-sends to the same domain on every request.
session tokensession token / session ID (giữ nguyên)An opaque random identifier naming one row in a server-side sessions table — meaningless by itself until looked up.
session storesession store (giữ nguyên) — bảng lưu session ở serverThe server-side table (or Redis) mapping session tokens to user ids and metadata — where session state actually lives, keeping handlers stateless.
JWTgiữ nguyên "JWT" — token tự chứa thông tin, ký bằng chữ ký sốJSON Web Token: a signed, self-contained blob of claims (user id, role, expiry) any server can verify by signature alone, no database lookup.
XSSgiữ nguyên "XSS" — chèn JS độc hại chạy trên trang của bạnCross-Site Scripting: an attacker's JavaScript runs on your page, able to read anything JS can read — including a token in localStorage.
CSRFgiữ nguyên "CSRF" — trang độc hại giả mạo request bằng cookie có sẵnCross-Site Request Forgery: a malicious page makes the victim's browser fire an authenticated request, since cookies attach automatically.
indexindex (giữ nguyên) — cấu trúc phụ giúp tìm nhanh, ghi chậm hơn chútAn extra on-disk structure letting the database jump to matching rows instead of scanning the whole table — at the cost of extra work on every write to that table.
selectivityselectivity (giữ nguyên) — cột lọc được bao nhiêu dữ liệuHow much a WHERE condition on a column narrows the result set. A near-unique column (email) is highly selective; a near-constant column (a 95%-true boolean) is not, and rarely benefits from an index.
N+1 query problemgiữ nguyên "N+1 query problem" — 1 query lấy danh sách rồi lặp N lần query thêmFetching a list (1 query) then looping over it firing one more query per item (N queries), instead of one JOIN or batched IN (...) lookup. No index fixes this — the access pattern itself is the bug.
transactiontransaction (giữ nguyên) — nhóm câu lệnh, hoặc tất cả cùng thành công hoặc tất cả cùng bị hủyA group of one or more database statements that either all take effect together, or none do — BEGIN, then COMMIT (keep everything) or ROLLBACK (discard everything).
isolationisolation (giữ nguyên) — mức độ transaction này bị ảnh hưởng bởi transaction khác chạy song songHow much a transaction is protected from the effects of OTHER transactions running at the same time. A separate guarantee from atomicity — "commits together" says nothing about what happens when two transactions interleave.
lost updategiữ nguyên "lost update" — 2 transaction cùng đọc rồi cùng ghi đè, một bên bị mất tác dụngA race condition where two concurrent transactions each read the same stale value, both decide to update based on it, and one write silently overwrites the other's effect instead of combining correctly.
row lockrow lock (giữ nguyên) — SELECT ... FOR UPDATE khóa dòng đó, transaction khác phải chờA SELECT ... FOR UPDATE clause that locks the returned row(s) until the transaction ends — another transaction trying to lock or update the same row waits its turn instead of racing ahead on stale data.
information disclosuregiữ nguyên "information disclosure" — lộ thông tin nội bộ qua response, giúp attacker vẽ bản đồ hệ thốngLeaking internal implementation details — stack traces, file paths, library versions, raw database errors — through an application's normal responses, giving an attacker a map of the system without needing to break in.
correlation ID / request IDgiữ nguyên "correlation ID" / "request ID" — mã nối một request với các dòng log server tương ứngA unique identifier generated for one incoming request, attached to every log line produced while handling it and safely returned to the client — the bridge that lets a support ticket be matched back to the exact server-side log entries, without exposing their contents.
structured logginggiữ nguyên "structured logging" — log dạng key-value/JSON, lọc/tìm chính xác theo fieldLogging as structured key-value data (often JSON) instead of free-text sentences, so fields like request_id, user_id, or error can be filtered and searched precisely across millions of log lines.
load balancergiữ nguyên "load balancer" — bộ phân phối request tới nhiều instanceA component that sits in front of multiple server instances and distributes incoming requests across them, usually round-robin or by least-connections.
horizontal scalinggiữ nguyên "horizontal scaling" — chạy thêm instance thay vì máy mạnh hơnScaling by adding more instances of a process running the same code, as opposed to vertical scaling which gives one instance more CPU/RAM.
Redisgiữ nguyên "Redis" — cache/key-value store dùng chung qua networkAn in-memory data store used as a shared cache or fast key-value store that every instance connects to over the network, instead of each instance keeping its own local copy.
shared-nothinggiữ nguyên "shared-nothing" — instance không giữ gì độc quyền, mọi state đều ở chỗ dùng chungAn architecture where no server instance holds any data that isn't also recoverable from a shared store, so any instance can be killed or replaced without losing anything.
sticky sessions / session affinitygiữ nguyên "sticky sessions" / "session affinity" — LB luôn route 1 client về đúng 1 instanceA load balancer configuration that routes a given client's requests to the same backend instance every time, instead of distributing freely.
cachecache (giữ nguyên) — nơi lưu tạm kết quả để tránh đọc/tính lạiA fast, temporary copy of data kept somewhere cheaper to read than its original source, so repeat reads skip the expensive origin — a DB query, a slow computation.
cache-asidecache-aside (giữ nguyên) — check cache trước, miss thì đọc DB rồi ghi lại vào cacheThe most common caching pattern: on read, check the cache first; on a miss, read the real source, then write that value into the cache before returning it — the next read for the same key is a hit.
cache hit / cache misscache hit/miss (giữ nguyên) — có sẵn trong cache hay khôngWhether a lookup found the value already in the cache (hit) or had to go to the original source and store it for next time (miss).
TTLTTL (giữ nguyên) — thời gian sống, hết hạn tự bị bỏTime-to-live: a cache entry's expiry duration; once it passes, the entry is dropped or ignored automatically, forcing the next read to refetch a fresh copy.
invalidate cacheinvalidate cache (giữ nguyên) — chủ động xóa cache khi data gốc đổiDeliberately removing or refreshing a cached entry the moment its underlying data changes, instead of only waiting for its TTL — needed for data that must never be stale right after a write.
stale readstale read (giữ nguyên) — đọc trúng cache cũ, gốc đã đổiA read that returns a cached value that no longer matches the current source of truth — the risk every cache introduces, traded for speed.
Cache-ControlCache-Control (giữ nguyên) — header HTTP nói được cache bao lâuAn HTTP response header telling browsers and intermediate proxies whether and how long they may cache a response themselves, with no application code involved.
