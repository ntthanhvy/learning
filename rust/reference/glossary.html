Rust course glossary

Every jargon term from the lessons: what it means in software (English), then the Vietnamese the way developers actually say it — mixed EN–VN, not textbook translation. Grows each day.

Day 1 — Cargo & the shape of Rust

TermIn softwareTiếng Việt (dev-style)
cargoRust's build tool and package manager in one: cargo new / check / run / test / clippy. You almost never call the compiler directly.giữ nguyên "cargo" — tool build + quản lý package
crateA Rust compilation unit / package — what other languages call a library or module you depend on. Binary crate = a program; library crate = code for others to use.giữ nguyên "crate" — như package/module
immutable by defaultlet x = 5 cannot be reassigned; you must opt in to change with let mut. The opposite default from Go.mặc định không đổi được, muốn sửa phải mut
shadowingDeclaring a new variable with the same name: let x = x + 1. The old one is gone; even the type may change. Legal and idiomatic in Rust.khai báo đè tên cũ bằng let mới
expression vs statementAlmost everything in Rust returns a value: an if, a block {}, a match. The last line of a block without ; IS its value.hầu hết mọi thứ đều trả về value, dòng cuối không ; là kết quả
enum (with data)A type that is exactly ONE of several variants, and each variant can carry its own data: Shape::Circle(f64). Go has nothing like this.giữ nguyên "enum" — mỗi variant mang được data riêng
matchCompare a value against patterns and run the matching arm. The compiler forces you to handle EVERY variant — forget one and it won't compile.giữ nguyên "match" — compiler bắt xử lý đủ mọi case
exhaustiveCovering every possible case, checked by the compiler. The reason match + enum eliminates a whole class of bugs.đủ hết mọi case, compiler kiểm tra
macroCode that writes code at compile time; called with !: println!, vec!. For now: "function-like thing with a bang".giữ nguyên "macro" — gọi có dấu !
impl blockWhere a type's methods live: impl Link { fn describe(&self) -> String {…} }. Rust's version of Go's receiver methods, gathered in one block.giữ nguyên "impl block" — nơi viết method cho type
variantOne of the listed shapes of an enum. Each variant can carry its own fields: Save { code, url } holds data, Help holds none.giữ nguyên "variant" — một nhánh của enum
clone.clone() makes an owned deep copy of a value. Cheap escape hatch from ownership fights early on; Day 2 explains when it's needed.giữ nguyên "clone" — copy riêng một bản
todo!() / matches!()todo!(): placeholder body that compiles but panics if reached. matches!(v, Pattern): true if the value has that shape — handy in tests.giữ nguyên — todo! giữ chỗ, matches! so shape

Day 2 — Ownership & borrowing

TermIn softwareTiếng Việt (dev-style)
ownershipEvery value has exactly one owner variable; when the owner goes out of scope the value is freed. This replaces the garbage collector.giữ nguyên "ownership" — mỗi value có đúng 1 chủ
moveAssigning or passing a value transfers ownership — the old variable becomes unusable, at compile time (error E0382). The compiler enforces it.giữ nguyên "move" — chuyển quyền sở hữu, biến cũ hết dùng được
Copy (trait)Marker for small stack-only types (i32, bool, char, f64): assignment duplicates instead of moving, both variables stay usable. String/Vec are NOT Copy.giữ nguyên "Copy" — gán là nhân bản, không move
borrow & / &mutTemporarily use a value without taking ownership. Many readers (&) OR one writer (&mut) — never both at once (error E0502 when violated).giữ nguyên "borrow" — mượn tạm, nhiều người đọc hoặc 1 người ghi
borrow checkerThe compiler pass that verifies the borrowing rules. Rust's famous gatekeeper: fights you early, saves you in production.giữ nguyên "borrow checker"
lifetimeHow long a reference is guaranteed valid. Mostly inferred; you'll read 'a long before you write it.giữ nguyên "lifetime" — reference sống được bao lâu
String vs &strString = owned, growable text (like a Go string builder you own). &str = a borrowed view into text someone else owns. Parameters borrow (&str), fields own (String).String là sở hữu, &str là mượn xem
slice &[T]A borrowed view into a sequence (part or all of a Vec or array). &Vec<T> auto-coerces to &[T] at call sites — accept slices in parameters.giữ nguyên "slice" — mượn xem một dãy

Day 3 — Option, Result & errors

TermIn softwareTiếng Việt (dev-style)
Option<T>The standard-library "maybe a value" enum: Some(v) or None. Replaces null entirely — match forces you to handle the empty case.giữ nguyên "Option" — thay hẳn cho null
if letA one-armed match: if let Some(x) = opt { … } runs only on the case you care about, skipping a do-nothing arm.giữ nguyên "if let" — match rút gọn, chỉ quan tâm 1 nhánh
Result<T, E>Success-or-failure enum: Ok(v) or Err(e), generic over both. Marked #[must_use] — the compiler warns if you silently drop one.giữ nguyên "Result" — như error value bên Go nhưng compiler ép kiểm tra
? operatorOn Err/None: return it from the current function immediately. On Ok/Some: unwrap and continue. Only legal in a function that itself returns Result/Option.toán tử ? — như if err != nil của Go gói trong 1 ký tự
thiserrorCrate providing #[derive(Error)]: writes Display + std::error::Error from #[error("...")] attributes. The library-code choice — name specific variants callers can match on.giữ nguyên "thiserror" — derive macro viết lỗi kiểu enum
anyhowCrate providing one universal anyhow::Error that any error converts into via ?. The application-code choice — propagate and print, don't match.giữ nguyên "anyhow" — kiểu lỗi chung ở tầng ứng dụng
lifetime 'aA name tying a returned reference's validity to an input's: fn f<'a>(x: &'a T) -> &'a T. Written explicitly only when a function both borrows and returns a borrow from it.giữ nguyên "lifetime" — buộc reference trả về không sống lâu hơn input

Day 4 — Traits, generics & iterators

TermIn softwareTiếng Việt (dev-style)
traitA named set of method signatures a type can implement. Unlike Go, satisfaction is never implicit — impl Trait for Type must be written explicitly, even for a type that already has a matching method.giữ nguyên "trait" — như interface bên Go nhưng khai báo tường minh
generics / trait boundA function written once over a type parameter T, constrained by a trait bound (T: Summary) naming what T must do. Resolved at compile time (monomorphization) — zero runtime cost.giữ nguyên "generics", ràng buộc bằng trait — resolve lúc compile
monomorphizationThe compiler's process of generating one specialized copy of a generic function per concrete type actually used, at compile time — why Rust generics cost nothing at runtime.giữ nguyên "monomorphization" — sinh bản riêng cho từng type lúc compile
dyn TraitDynamic dispatch: the concrete type is decided at runtime via a vtable pointer, not compile time. Used for mixed collections where the type genuinely varies — closer to how Go interface values always work.giữ nguyên "dyn Trait" — quyết định type lúc runtime qua vtable
closureAn anonymous, inline function that can capture variables from its surrounding scope: |l| l.code == code. Iterator adapters take closures to describe per-element work.giữ nguyên "closure" — hàm nặc danh bắt biến xung quanh
iterator adapterChainable, lazy methods on anything implementing Iterator — .find(), .map(), .filter(), .collect(). Nothing runs until the chain is consumed.chuỗi find/map/filter/collect, chạy lazy tới khi bị tiêu thụ

Day 5 — Collections, modules & serde

TermIn softwareTiếng Việt (dev-style)
moduleA named unit of code organization forming a tree, declared with mod. A file can BE a module (src/link.rs, declared with mod link;) or a module can be inline. Boundaries are explicit — nothing crosses one by default, unlike Go's per-directory package.giữ nguyên "module" — đơn vị tổ chức code dạng cây, ranh giới rõ ràng
pubVisibility modifier making an item (struct, field, function, module) accessible outside the module defining it. Default is private, per-item — the opposite of Go's capitalized-name-means-exported convention.giữ nguyên "pub" — cho phép item nhìn thấy từ ngoài module, mặc định private
HashMapHashMap<K, V> — average O(1) insert/lookup/remove by key, unlike Vec's O(n) linear scan. The same data structure as Go's built-in map; iteration order is unspecified.giữ nguyên "HashMap" — bảng băm, tra cứu theo key trung bình O(1), giống map của Go
serdeTHE Rust (de)serialization framework. #[derive(Serialize, Deserialize)] generates real, compile-time-checked code converting a struct to/from JSON (via serde_json) — no runtime reflection.giữ nguyên "serde" — framework chuyển struct sang/từ JSON bằng derive macro
deriveAn attribute like #[derive(Trait)] asking the compiler to auto-generate a standard trait implementation, instead of writing it by hand. Used for Serialize/Deserialize, Debug, Clone, and more.giữ nguyên "derive" — compiler tự viết impl chuẩn, không cần code tay

Day 6 — Async & tokio

TermIn softwareTiếng Việt (dev-style)
futureAsync functions in Rust are lazy: calling one builds a value implementing Future<Output = T> and runs nothing until that future is polled — by .await, or by handing it to an executor like tokio::spawn. Unlike a goroutine, nothing runs on its own.giữ nguyên "future" — đại diện việc chưa xong, không tự chạy cho tới khi được await/spawn
.awaitThe point where an async function can pause, handing control back to whatever is driving it, and resume once the awaited future is ready. The underlying thread is free to run other work while paused — not a blocking call.giữ nguyên "await" — điểm hàm async tạm dừng, nhường CPU, chạy tiếp sau
tokioThe de-facto standard async runtime for Rust — the scheduler that actually polls futures to completion, plus async IO, timers, and channels. Rust's stdlib ships no runtime of its own; tokio is a library choice, not part of the language.giữ nguyên "tokio" — runtime chuẩn chạy future, kèm IO/timer bản async
taskA unit of async work spawned via tokio::spawn — NOT an OS thread. Many tasks are scheduled cooperatively onto a small pool of OS threads, the same M:N idea Go's scheduler uses for goroutines, as a library instead of a language runtime.giữ nguyên "task" — đơn vị công việc async, nhiều task chạy trên ít OS thread, giống mô hình goroutine của Go
SendA marker trait meaning a value is safe to move to another thread. Automatically implemented by the compiler when every field of a type is itself Send — no manual derive needed for ordinary owned data like String or Vec.giữ nguyên "Send" — an toàn khi chuyển giá trị sang thread khác
SyncA marker trait meaning a shared reference (&T) is safe for multiple threads to hold at once. Also automatically derived when safe; a type with unsynchronized interior mutability would need something like Mutex to become Sync.giữ nguyên "Sync" — an toàn khi nhiều thread cùng đọc qua tham chiếu
'static bound (spawned tasks)A future handed to tokio::spawn can't borrow anything from its spawning function — it may run later, on another thread, after that function has returned. The fix is to own the data (move it in), not borrow it.giữ nguyên "'static" — task spawn phải sở hữu dữ liệu, không mượn tham chiếu

Day 7 — Capstone: an axum service

TermIn softwareTiếng Việt (dev-style)
handlerIn axum, an ordinary async fn whose parameters are types implementing an extractor trait and whose return type implements IntoResponse — no special base trait to implement by hand.giữ nguyên "handler" — hàm async xử lý 1 request
extractorA type implementing axum's request-extraction trait, letting a handler declare exactly what part of the request it needs (Path, Json, State) as an ordinary typed parameter instead of parsing the raw request by hand.giữ nguyên "extractor" — khai báo thẳng phần request cần dùng làm tham số hàm
IntoResponseaxum's trait for anything convertible into an HTTP response. Implementing it for a custom error type lets that type be the Err side of a handler's Result, with axum calling into_response() automatically when ? returns it early.giữ nguyên "IntoResponse" — trait chuyển 1 giá trị Rust thành HTTP response thật
ArcAtomic reference-counted pointer: multiple owners of the same heap value, each a cheap-to-clone handle; freed once the last owner drops it. The thread-safe cousin of Rc, required whenever shared data crosses task/thread boundaries.giữ nguyên "Arc" — con trỏ đếm tham chiếu, nhiều nơi cùng sở hữu 1 giá trị, an toàn giữa các thread
MutexA mutual-exclusion lock: only one task can hold it (via .lock()) at a time; any other locker blocks until release, guaranteeing two concurrent requests never mutate shared data at the same instant.giữ nguyên "Mutex" — khóa loại trừ, chỉ 1 nơi được sửa dữ liệu tại một thời điểm
State extractorState<S> hands a handler the shared application state (here, the Arc<Mutex<HashMap>>) registered on the router via .with_state(...) — axum's dependency-injection mechanism for handlers.giữ nguyên "State" — lấy state dùng chung đã đăng ký ở router

Coming this week (Days 3–7)

TermIn softwareTiếng Việt (dev-style)
Option<T>"Maybe a value": Some(v) or None. Rust has no null — this type replaces it, and match forces you to handle the None case.giữ nguyên "Option" — thay cho null
Result<T, E>"Success or failure": Ok(v) or Err(e). Errors are values (like Go!), but the compiler won't let you ignore them.giữ nguyên "Result" — như error value bên Go nhưng compiler bắt xử lý
? operator"If this is Err/None, return it to my caller now; otherwise unwrap and continue." Go's if err != nil {return err} in one character.toán tử ? — như if err != nil của Go gói trong 1 ký tự
traitA named set of behavior a type can implement — Rust's version of a Go interface, but implementations are declared explicitly with impl Trait for Type.giữ nguyên "trait" — như interface bên Go nhưng khai báo tường minh
derive#[derive(Debug, Clone)] — ask the compiler to auto-write standard trait implementations for your type.giữ nguyên "derive" — compiler tự viết impl chuẩn
generics / trait boundCode over any type T, constrained by what it can do: fn f<T: Display>(x: T). Resolved at compile time — no runtime cost.giữ nguyên "generics", ràng buộc bằng trait
closureAn anonymous function that can capture variables around it: |x| x + 1.giữ nguyên "closure" — hàm nặc danh bắt biến xung quanh
iterator adapterChainable lazy operations on sequences: .map(...).filter(...).collect(). Nothing runs until collect/consumption.chuỗi map/filter/collect, chạy lazy
serde / (de)serializationTHE Rust framework for converting structs to/from JSON and friends, via #[derive(Serialize, Deserialize)].giữ nguyên "serde" — struct ⇄ JSON
async / awaitWrite non-blocking code that reads like blocking code. An async fn returns a future; .await is where it can pause.giữ nguyên "async/await"
futureA value representing work not finished yet. Does NOTHING until awaited — unlike a goroutine, which runs immediately.giữ nguyên "future" — chưa chạy cho tới khi await
tokioThe standard async runtime: the scheduler that actually runs futures, plus async IO, timers, tasks. Rust ships no runtime of its own.giữ nguyên "tokio" — runtime chạy async
axumThe de-facto standard web framework (by the tokio team): routing, extractors, middleware via tower.giữ nguyên "axum" — web framework chuẩn hiện nay
clippyThe official linter: cargo clippy. Like go vet plus hundreds of idiom hints.giữ nguyên "clippy" — linter chính chủ
