Basics — declarations, pointers & construction

TermIn softwareTiếng Việt (dev-style)
zero valueThe default a declared variable gets with no initializer: 0, "", false; nil for pointers, maps, slices, interfaces, channels.giá trị mặc định khi khai báo
short declaration :=Declare and assign in one step, type inferred. Only inside functions.khai báo nhanh, tự suy kiểu
composite literalT{...} — create a struct/slice/map value and fill its contents in one expression. Unset struct fields get zero values.tạo value kèm nội dung: T{...}
pointer *TA value holding the address of another value; lets two places share ONE value instead of copies. In a type position, *T reads "pointer to T."con trỏ
address-of &&v gives a pointer to v. &T{...} creates the value AND takes its address in one step.lấy địa chỉ, trả về con trỏ
dereference *pFollow the pointer to the value it points at. With fields Go auto-dereferences: p.Field just works.đọc value mà con trỏ trỏ tới
receiverThe (s *MemStore) part of a method — what the method is called on. Pointer receiver shares the original; value receiver gets a copy.giữ nguyên "receiver"
make vs newmake initializes maps/slices/channels ready for use; new(T) returns a zeroed *T — in practice &T{} is preferred.make cho map/slice/chan; new hiếm dùng
nilThe "points at nothing / not yet set" value of pointers, maps, slices, interfaces, channels, funcs. Writing to a nil map panics.giữ nguyên "nil"
panicThe program stops immediately and prints a stack trace — a crash, distinct from returning an error value. Idiomatic Go returns errors; panics are for impossible states.giữ nguyên "panic"

Day 1 — Interfaces & composition

TermIn softwareTiếng Việt (dev-style)
interfaceA named list of methods. Any type that has those methods can be used wherever the interface is asked for.giữ nguyên "interface"
satisfyTo have every method an interface lists, so the compiler accepts the type as that interface. Automatic — nothing declared.thỏa mãn / "có đủ method"
implicitHappens automatically without you writing anything to declare it. Opposite: explicit.ngầm định, tự động
consumer / implementerConsumer = the code that calls the behavior (receives the interface). Implementer = the code that provides the method bodies.bên dùng / bên hiện thực
concrete typeA real struct or named type — not an interface. All fields and methods visible.kiểu cụ thể (struct thật)
abstractionHiding details so callers deal only with what matters — an interface is an abstraction over concrete types.trừu tượng hóa
compositionBuilding bigger things by combining small parts, instead of inheriting from a parent class.ghép từ nhiều phần nhỏ (vs. kế thừa)
embeddingDeclaring a field with only a type, no name, inside a struct. The inner type's methods become usable on the outer type.nhúng type vào struct
promoted methodA method "lifted" from the embedded value to the outer type: outer.Method() forwards automatically.method bên trong được gọi thẳng trên type ngoài
delegateTo pass the call through to the wrapped inner value, which does the real work — e.g. s.Store.Save(ctx, l).chuyển việc cho bên trong xử lý
wrapper / wrapA type holding another value inside, adding behavior around its calls (logging, caching, retries).giữ nguyên "wrap" — bọc thêm hành vi
decoratorA wrapper that changes ONE behavior and delegates everything else. Same idea as middleware.giữ nguyên "decorator"
middlewareA layer wrapped around HTTP request handling that adds one concern (logging, auth, rate limits) then passes the request on. Built hands-on in Day 4.giữ nguyên "middleware"
idiomaticWritten the way experienced Go programmers expect — not just "works," but "looks right" in code review.đúng "chuẩn" của cộng đồng Go
scaffoldTo create a project's first skeleton: folders, module, empty types — the frame you build on.dựng khung ban đầu
test fakeA tiny hand-written stand-in (e.g. a fake Store) used in tests instead of the real implementation.object giả để test

Day 2 — Errors as values

TermIn softwareTiếng Việt (dev-style)
error (interface)Not a special mechanism — an ordinary one-method interface: Error() string. Errors are values you return, compare, and store.giữ nguyên "error" — chỉ là interface một method
sentinel errorA single package-level pre-declared error value (like ErrNotFound) that callers compare against with errors.Is. Named with the Err prefix.giữ nguyên "sentinel error" — error khai báo sẵn để so sánh
wrap (an error) / %wAttach context around an error with fmt.Errorf("…: %w", err), keeping the original inside and findable through the layers.giữ nguyên "wrap" — bọc error, error gốc vẫn bên trong
error chainThe nested sequence wrapping creates: outer context first, root cause innermost. errors.Is/As walk it; errors.Unwrap exposes one step.chuỗi error bọc nhau, đi từ ngoài vào gốc
errors.Is"Is this specific error value anywhere in the chain?" — the correct replacement for == once wrapping exists.check error CÓ trong chain không (thay cho ==)
errors.As"Is there an error of this TYPE in the chain?" — finds it and copies it out so you can read its fields (e.g. a DB error code).tìm error theo TYPE trong chain rồi lấy data ra
handle onceExactly one layer decides what an error means (404? retry? crash?). Layers below only add context and return — never log-and-return.xử lý error đúng một nơi, không vừa log vừa return
comma-okThe two-value read v, ok := m[k] — ok says whether the key existed, separating "missing" from "zero value stored".đọc map 2 giá trị, ok báo key có tồn tại không

Day 3 — Goroutines, channels, select

TermIn softwareTiếng Việt (dev-style)
concurrency / parallelismConcurrency = structuring work as independent tasks. Parallelism = tasks literally running at the same instant. Go gives you the first; the runtime supplies the second.đồng thời / song song
goroutineA function running concurrently, managed by the Go runtime — a few KB each. Started with go f(); all die instantly when main returns.giữ nguyên "goroutine"
WaitGroupCounter from sync: Add(1) before launching, defer Done() inside, Wait() blocks until zero — the standard "wait for all workers" tool.giữ nguyên "WaitGroup" — bộ đếm chờ xong hết
channelTyped pipe between goroutines: ch <- v sends, <-ch receives. Unbuffered = send and receive wait for each other (sync point); buffered = sends queue until full.giữ nguyên "channel"
close (a channel)Sender's signal that no more values are coming. Receivers detect it via comma-ok: r, open := <-ch, open == false when closed and drained.giữ nguyên "close" — báo hết dữ liệu
selectswitch for channels: blocks until one case can proceed, random pick if several are ready. The "result, timeout, or cancel — whichever first" tool.giữ nguyên "select" — chờ nhiều channel, cái nào xong trước chạy
data raceTwo goroutines touching the same data at once, at least one writing — the result depends on scheduling luck. Found by go test -race.giữ nguyên "data race" — tranh chấp dữ liệu
mutexMutual-exclusion lock (sync.Mutex): Lock()/Unlock() so only one goroutine touches shared data at a time.giữ nguyên "mutex" — khóa dữ liệu chung
race detectorThe -race flag: instruments memory accesses and prints both guilty stack traces when a data race happens during tests or runs.giữ nguyên "race detector" — cờ -race
fan-out / fan-inFan-out: launch N workers for N tasks. Fan-in: collect their results into one channel. Today's checkAll is both.tỏa ra nhiều worker / gom kết quả về một mối

Day 4 — context, sync & the HTTP server

TermIn softwareTiếng Việt (dev-style)
context.ContextA value carrying a cancellation signal and optional deadline, passed as the first argument down a whole call chain. ctx.Done() closes when work should stop; ctx.Err() says why.giữ nguyên "context" — tín hiệu hủy/deadline truyền xuyên suốt
deadlineThe specific point in time a context is cancelled at. context.WithTimeout(ctx, d) is shorthand for deadline = now + d.giữ nguyên "deadline" — mốc thời gian bị hủy
goroutine leakA goroutine blocked forever on a channel op that will never happen, so it's never collected. Small ones are invisible; under load they exhaust memory. Fixed by always giving blocking ops a ctx.Done() exit.giữ nguyên "goroutine leak" — goroutine kẹt mãi mãi
wildcard segmentA named path segment in a Go 1.22+ ServeMux route ("GET /{code}"), read in the handler via r.PathValue("code"). No third-party router required for this anymore.giữ nguyên "wildcard segment" — phần path đặt tên trong route
graceful shutdownStop accepting new connections, let in-flight requests finish, then exit. http.Server.Shutdown(ctx) implements it, blocking until every handler returns or ctx expires.giữ nguyên "graceful shutdown" — tắt server: ngừng nhận request mới, chờ request cũ xong rồi thoát
errgroupgolang.org/x/sync/errgroup: launch N goroutines, cancel all of them the moment one errors, wait for the rest — the production upgrade of a plain WaitGroup fan-out.giữ nguyên "errgroup"

Day 5 — Data modeling in PostgreSQL

TermIn softwareTiếng Việt (dev-style)
invariantA condition that must ALWAYS hold, no matter what code runs — e.g. "every click points to an existing link." Constraints are how the database enforces invariants itself.giữ nguyên "invariant" — điều kiện luôn luôn đúng, DB tự ép
constraintA rule the database enforces on every write, from every caller: PRIMARY KEY, NOT NULL, UNIQUE, CHECK, FOREIGN KEY. Beats trusting every code path to remember the rule.giữ nguyên "constraint" — luật DB tự ép, không cần code nhớ
normalization / denormalizationNormalized: each fact stored once, derived facts computed by query. Denormalized: a fact deliberately duplicated/pre-computed for read speed — a trade-off that must be justified, not a default.chuẩn hóa (mỗi sự thật 1 chỗ) / giữ nguyên "denormalization" (cố ý lưu trùng để đọc nhanh)
foreign keyA column holding another table's primary key, enforcing that every row here points at a row that actually exists there — no dangling references, checked on every write.giữ nguyên "foreign key" (FK) — cột trỏ sang khóa chính bảng khác
natural keyA primary key that is itself an already-unique business value (linkshort's code) — no extra column needed.giữ nguyên "natural key" — khóa chính là giá trị nghiệp vụ có sẵn
surrogate keyA primary key manufactured purely to identify a row, no business meaning — an auto-incrementing integer (GENERATED ALWAYS AS IDENTITY) or UUID. Used when nothing is naturally unique.giữ nguyên "surrogate key" — khóa chính tự sinh, không mang ý nghĩa nghiệp vụ
B-tree indexPostgres's default index structure: lets the database jump to matching rows instead of scanning the whole table. Good for equality, ranges, and ORDER BY on the indexed column(s).giữ nguyên "B-tree" — cấu trúc index mặc định, tốt cho so sánh bằng/khoảng
composite indexAn index over more than one column, ordered left-to-right. Serves queries filtering on the first column alone, or the first AND second together — not the second alone.giữ nguyên "composite index" — index gộp nhiều cột, thứ tự quan trọng
partial indexAn index built over only the rows matching a WHERE clause — smaller and cheaper when just a hot subset of a big table is ever queried.giữ nguyên "partial index" — chỉ đánh index một phần bảng thỏa điều kiện
migration (DB)A numbered, checked-in SQL script that changes the schema by one step. Applying migrations in order reproduces the exact same schema in any environment.giữ nguyên "migration" — file SQL đánh số, chạy lại được ở mọi môi trường

Day 6 — Queries from Go (+ generics in the wild)

TermIn softwareTiếng Việt (dev-style)
connection poolA small set of already-open database connections, reused across calls instead of opening a new TCP + auth handshake per query. *pgxpool.Pool is the pgx v5 pool type.giữ nguyên "connection pool" — giữ sẵn vài kết nối DB, dùng lại thay vì mở mới
genericA function or type written once, parameterized over a type argument T, that works for any T satisfying its constraints — instead of one hand-copied version per concrete type. pgx.CollectRows[T] is a real-world use.giữ nguyên "generic" — viết một lần, dùng cho nhiều kiểu T khác nhau
offset paginationReturning results page by page by skipping a row count from the start (LIMIT m OFFSET n). Simple, but gets slower the deeper you page — Postgres still walks and discards the first n rows every time.giữ nguyên "offset pagination" — phân trang bằng cách bỏ qua n dòng đầu
keyset paginationPagination using a WHERE clause on the last row's own sort key instead of a row count. Cost stays flat no matter how deep you page — exactly what a composite index like clicks (link_code, clicked_at DESC) is built to serve.giữ nguyên "keyset pagination" — phân trang bằng điều kiện WHERE trên giá trị dòng cuối
transactionA group of statements that succeed or fail together as one unit — BEGIN, then COMMIT (all changes take effect) or ROLLBACK (all discarded). Needed once a unit of work spans more than one statement; a single statement is already atomic.giữ nguyên "transaction" — nhóm câu lệnh, hoặc tất cả cùng thành công hoặc tất cả cùng bị hủy
query plan / EXPLAINThe step-by-step strategy Postgres actually picked to answer a query — which index (if any), which scan type. EXPLAIN ANALYZE runs the query for real and shows both the plan and real timings.giữ nguyên "query plan" — kế hoạch Postgres chọn để chạy query, xem bằng EXPLAIN

Day 7 — Testing, tooling & capstone

TermIn softwareTiếng Việt (dev-style)
table-driven testOne test function that loops over a slice of input/expected-output cases instead of one hand-written function per case — the standard shape for Go tests with more than one or two cases.giữ nguyên "table-driven test" — một hàm test, chạy qua danh sách case
subtestA named, independently-reported test running inside a parent test via t.Run(name, func(t *testing.T) {...}). Failures show up per-subtest name; go test -run can target one by name.giữ nguyên "subtest" — test con bên trong một hàm test, chạy được riêng lẻ
integration testA test exercising real collaborating systems together (here: actual SQL against a real Postgres database) instead of a fake/in-memory stand-in. Slower, needs real infrastructure, catches bugs a unit test structurally cannot.giữ nguyên "integration test" — test chạy với hệ thống thật, không phải bản giả lập
build tagA //go:build tag comment above the package clause that excludes a file from a normal go build/go test unless that tag is explicitly passed with -tags. Keeps slow or infrastructure-dependent tests out of the default run.giữ nguyên "build tag" — dòng comment loại file khỏi build/test mặc định, trừ khi truyền -tags
benchmarkA test function measuring how fast code runs rather than whether it's correct — named BenchmarkX, run with go test -bench=Pattern. Go 1.24's b.Loop() is the modern iteration shape, replacing the older for i := 0; i < b.N; i++.giữ nguyên "benchmark" — hàm đo tốc độ chạy code, không kiểm tra đúng/sai
modernizer (go fix)A tool that rewrites code to a newer, equivalent idiom it's certain about — e.g. a hand-written min/max helper collapsed into the language builtin. Never changes behavior, only style.giữ nguyên "modernizer" — công cụ tự viết lại code theo lối viết mới hơn, không đổi hành vi
pprofGo's built-in profiler: go test -bench=. -cpuprofile=cpu.out then go tool pprof cpu.out turns "it feels slow" into an actual ranked list of where the CPU time went.giữ nguyên "pprof" — công cụ đo hiệu năng có sẵn của Go
