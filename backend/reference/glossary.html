Backend Foundations · Reference

Glossary

Every jargon term used in the lessons, in one place. Tiếng Việt column is dev-style — how Vietnamese devs actually say it, not textbook translation.

TermTiếng Việt (dev-style)In software
serverserver (giữ nguyên)A long-running program that listens on a port and answers requests. Not the physical machine — the process.
portport / cổng (thường giữ nguyên)A number (like 8080 or 5432) that lets one machine run many listening programs; the address within a machine.
endpointendpoint (giữ nguyên)One callable URL + method that an API exposes, e.g. GET /api/orders/{id}.
route / routingroute (giữ nguyên); routing = định tuyếnThe rule mapping a method + path pattern to the function that should handle it; routing is the act of matching.
handlerhandler (giữ nguyên) — hàm xử lý requestThe function that receives one request and produces one response. The backend equivalent of an event handler.
middlewaremiddleware (giữ nguyên) — lớp chạy trước/sau handlerCode wrapped around handlers to do cross-cutting work — logging, auth checks, timeouts — so every handler doesn't repeat it.
business logiclogic nghiệp vụ / business logicThe rules of your product ("an order over $100 ships free"), kept separate from HTTP parsing and database code.
statelessstateless (giữ nguyên) — không giữ trạng thái giữa các requestThe server keeps nothing in memory between requests; each request starts from zero. State that must survive goes to the database.
persistencepersist / lưu xuống DBStoring data so it survives after the process stops — usually meaning "write it to the database".
connection poolconnection pool (giữ nguyên) — pool kết nối DBA kept-warm set of database connections that handlers borrow and return, because opening a fresh connection per request is too slow.
serializationserialize (giữ nguyên) — chuyển object thành JSON/bytesTurning an in-memory value into bytes to send over the wire (e.g. struct → JSON); deserialization is the reverse.
schemaschema (giữ nguyên) — cấu trúc bảng của DBThe set of tables, columns, and rules defining what data can exist — the database's shape, shared by every feature.
entityentity (giữ nguyên) — thực thể, có id riêngA thing in the product with its own identity and lifecycle — it gets an id, usually its own table, and other things refer to it.
primary keyprimary key / khóa chính (hay nói tắt PK)The column that uniquely identifies one row — usually an id — which other tables point at.
foreign keyforeign key / khóa ngoại (hay nói tắt FK)A column holding another table's primary key, expressing "this row belongs to that row".
join tablejoin table / bảng trung gianA table whose rows each connect two other tables' rows — how a many-to-many relationship is stored (e.g. order_items between orders and products).
normalizationnormalization / chuẩn hóa dữ liệuStructuring tables so each fact is stored exactly once and everything else references it by key. Instinct version: one fact, one place.
denormalizationdenormalization (giữ nguyên) — cố ý lưu trùng dữ liệuDeliberately storing a fact in more than one place — a trade-off you must justify (e.g. snapshotting a price onto an order line).
API contractAPI contract (giữ nguyên) — hợp đồng giữa backend và clientThe agreed shape of an API — URLs, verbs, bodies, error format. The promise frontends build against; breaking it breaks them.
resourceresource (giữ nguyên) — "danh từ" mà API exposeA thing clients address by URL — usually an entity or collection (/orders, /orders/42). Verbs act on it; you don't make /getX endpoints.
idempotentidempotent (giữ nguyên) — gọi N lần như gọi 1 lầnSame end state whether performed once or many times, so blind retries are safe. GET/PUT/DELETE promise it; POST does not.
status codestatus code (giữ nguyên) — chữ số đầu nói ai có lỗiThe response's three-digit result. 2xx worked; 4xx the client's request was wrong (fixable by caller); 5xx the server broke (alarm).
error contracterror contract — MỘT format lỗi thống nhấtThe single JSON error shape every failing response uses, with a stable machine-readable code clients branch on — the API's sentinel errors.
authenticationauthentication (giữ nguyên) — xác minh "bạn là ai"Proving who is making a request — distinct from authorization, which decides what they're allowed to do once known.
cookiecookie (giữ nguyên)A small piece of data the browser stores and automatically re-sends to the same domain on every request.
session tokensession token / session ID (giữ nguyên)An opaque random identifier naming one row in a server-side sessions table — meaningless by itself until looked up.
session storesession store (giữ nguyên) — bảng lưu session ở serverThe server-side table (or Redis) mapping session tokens to user ids and metadata — where session state actually lives, keeping handlers stateless.
JWTgiữ nguyên "JWT" — token tự chứa thông tin, ký bằng chữ ký sốJSON Web Token: a signed, self-contained blob of claims (user id, role, expiry) any server can verify by signature alone, no database lookup.
XSSgiữ nguyên "XSS" — chèn JS độc hại chạy trên trang của bạnCross-Site Scripting: an attacker's JavaScript runs on your page, able to read anything JS can read — including a token in localStorage.
CSRFgiữ nguyên "CSRF" — trang độc hại giả mạo request bằng cookie có sẵnCross-Site Request Forgery: a malicious page makes the victim's browser fire an authenticated request, since cookies attach automatically.
