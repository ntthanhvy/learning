Rust Intensive · Day 1 · ~75 min

Cargo & the shape of Rust

Your first Rust program is the linkshort project again — this time you'll meet the two things Go simply doesn't have: enums that carry data, and a compiler that refuses to let you forget a case.

Today's win: a working linkshort-rs CLI — save, get, help — where the command itself is an enum and the compiler proves you handled every case.

1 · Cargo in five commands

In Go you juggled go mod init, go vet, go test, go run. Rust rolls all of it into cargo:

CommandDoesGo equivalent
cargo new xnew project (a crate) with git + foldersmkdir + go mod init
cargo checkcompile-check, no binary — fast, use constantlygo vet-ish
cargo runbuild + run (-- args after the --)go run .
cargo testrun all testsgo test ./...
cargo clippythe official linter, with idiom hintsgo vet + more

Project layout after cargo new: Cargo.toml (like go.mod — name, version, dependencies) and src/main.rs (like main.go). That's it.

2 · let, mut, and "everything is an expression"

let x = 5;          // immutable — `x = 6` later is a COMPILE ERROR
let mut y = 5;      // opt in to change — the reverse of Go's default
let x = x + 1;      // legal! a NEW x replaces the old one (even a new type)

Line three is shadowing — idiomatic Rust, where Go would refuse.

The deeper shift: almost everything is an expression — it produces a value. Go's if does things; Rust's if is a value:

let label = if count > 0 { "some" } else { "none" };  // if as a value (no ternary needed)

fn double(n: i32) -> i32 {
    n * 2               // last line, NO semicolon → this IS the return value
}

Add a semicolon to n * 2 and it becomes a statement returning nothing — the compiler will complain the function returns (). This "missing semicolon as return" is everywhere in real Rust; read it as the block's final value (Book ch. 3.3).

3 · Structs and impl

Structs look like Go. Methods don't live on receivers — they live in an impl block:

struct Link {
    code: String,
    url: String,
}

impl Link {
    fn describe(&self) -> String {              // &self ≈ Go's (l *Link) receiver — Day 2 explains the &
        format!("{} -> {}", self.code, self.url) // format! builds a String, like fmt.Sprintf
    }
}

let l = Link { code: String::from("rs"), url: String::from("https://rust-lang.org") };
println!("{}", l.describe());

println! and format! end in ! because they're macros, not functions. For now: a function-like thing with a bang.

4 · The star: enums with data + match

In Go, "one of several kinds of thing" meant iota constants, or an interface, or a struct with unused fields — and nothing checked you covered every kind. Rust makes it a first-class type: each variant can carry its own data:

enum Command {
    Save { code: String, url: String },  // this variant carries two fields
    Get { code: String },                // this one carries one
    Help,                                // this one carries nothing
}

A Command is exactly one of these shapes. And match takes them apart, binding the fields as it goes:

match cmd {
    Command::Save { code, url } => links.push(Link { code, url }),
    Command::Get { code } => println!("looking up {code}"),
    Command::Help => println!("usage: …"),
}

The contract: match must be exhaustive. Forget a variant and the build fails — you'll trigger this on purpose in the practice. Adding a fourth variant next month? Every match in the codebase becomes a compile error until it's handled. That's the feature backend teams rave about (Book ch. 6).

Step 2 — the skeleton. Replace the whole of src/main.rs with this. Everything is given except three TODOs — read the given parts, they use today's sections:

    code: String,
    url: String,
}

    // no `return`, no trailing `;`. Until then, todo!() keeps the compiler happy.
    fn describe(&self) -> String {
        todo!()
    }
}

// TODO 1: define `enum Command` with three variants:
//   Save { code: String, url: String }   — save a new link
//   Get  { code: String }                — look one up
//   Help                                 — no data at all
// (This is the heart of today. Write it, then run `cargo check`.)

// GIVEN: turns CLI words into a Command. `match` on a slice: each pattern
// describes a SHAPE of the arguments ([three words], [two words], anything else).
fn parse_command(args: &[String]) -> Command {
    match args {
        [cmd, code, url] if cmd == "save" => Command::Save {
            code: code.clone(), // .clone(): make an owned copy — the why is Day 2
            url: url.clone(),
        },
        [cmd, code] if cmd == "get" => Command::Get { code: code.clone() },
        _ => Command::Help,
    }
}

fn main() {
    let args: Vec<String> = env::args().skip(1).collect();
    let cmd = parse_command(&args); // &: lend it, don't give it away — Day 2's topic

    let mut links = vec![
        Link { code: String::from("go"), url: String::from("https://go.dev") },
        Link { code: String::from("rust"), url: String::from("https://www.rust-lang.org") },
    ];

    // TODO 2: `match cmd { … }` — step 5 walks you through it.
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn empty_args_means_help() {
        let cmd = parse_command(&[]);
        assert!(matches!(cmd, Command::Help)); // matches!: "does cmd have this shape?"
    }

    #[test]
    fn save_parses_code_and_url() {
        let args = vec![
            "save".to_string(),

Step 3 — TODO 1, the enum. Write enum Command where the TODO 1 comment sits (the comment tells you the three variants exactly). Then:

cargo check

It should pass, with ~7 warnings about unused things — fine, the code that uses them is coming. If you see error instead, read it: the compiler names the line and usually suggests the fix. Rust's errors are the best in the business — reading them is a skill this course trains on purpose.

Step 4 — TODO 3, the method. Replace todo!() in describe with a format! expression producing go -> https://go.dev style output. No return, no semicolon (section 2!). cargo check again — still passing?

Step 5 — TODO 2, the match. Where the TODO 2 comment sits, start with only one arm, deliberately incomplete:

match cmd {
    Command::Help => {
        println!("usage: linkshort-rs save <code> <url> | get <code>");
    }
}

Run cargo check and read the failure — this is today's lesson in one error:

error[E0004]: non-exhaustive patterns: `Command::Save { .. }` and `Command::Get { .. }` not covered

Go would have shipped that bug. Now add the two missing arms — Save pushes and lists, Get searches:

Command::Save { code, url } => {
    links.push(Link { code, url });
    println!("saved. {} links now:", links.len());
    for l in &links {
        println!("  {}", l.describe());
    }
}
Command::Get { code } => {
    let mut found = false;
    for l in &links {
        if l.code == code {
            println!("{}", l.describe());
            found = true;
        }
    }
    if !found {
        println!("no link with code '{}'", code);
    }
}

Step 6 — prove it.

cargo check          # quiet — zero warnings now
cargo test           # test result: ok. 2 passed
cargo run -- get rust
cargo run -- save cc https://claude.com
cargo run            # no args → usage line

Expected: get rust prints rust -> https://www.rust-lang.org; the save prints saved. 3 links now: and lists all three; no args prints usage. (Links vanish between runs — it's an in-memory Vec, exactly like your Go MemStore before Day 5.)

Done when: cargo check quiet · cargo test shows 2 passed · the three runs above print what they should. Bonus polish: cargo clippy stays silent too.
